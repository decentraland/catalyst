# Catalyst

[![Coverage Status](https://coveralls.io/repos/github/decentraland/catalyst/badge.svg?branch=main)](https://coveralls.io/github/decentraland/catalyst?branch=main)

A Catalyst is a server that bundles different services. These services form the backbone of Decentraland, providing decentralized storage for most of the content needed by clients.

If you just want to run a Catalyst server, please check the [Catalyst Owner](https://github.com/decentraland/catalyst-owner) repository. This repository is primarily intended for development.

## Features

- **Sync**: Identifies itself and other nodes from the DAO list (official Catalysts list) and periodically polls other nodes to replicate their deployments to its own file system
- **Entity Deployment**: Acts as a content server to store every entity used in the Decentraland ecosystem
- **Entity Retrieval**: Provides endpoints to retrieve raw entities (via Content Server) and sanitized entities (via Lambdas, which validates ownership against the blockchain)
- **Failed Deployments Retrieval**: Provides an endpoint to fetch deployments that have failed on the node
- **Content Retrieval**: Provides endpoints to retrieve raw binary files associated with deployed entities
- **Snapshots Retrieval (sync)**: Provides an endpoint to retrieve snapshots generated by the node, which contain all historical deployments that have happened within a specific time frame
- **Collections Retrieval**: Provides endpoints to retrieve information about item collections
- **Users Items Retrieval**: Provides endpoints to retrieve a user's wearables, emotes, and outfits (off-chain and on-chain items, including linked NFTs)
- **DAO List Fetch**: Provides endpoints to retrieve nodes participating in the Catalyst network (nodes listed in the DAO list that officially replicate content)

## Dependencies & Related Services

Each Catalyst node depends only on other official nodes (from the DAO list) for content replication.

The [Catalyst-Client](https://github.com/decentraland/catalyst-client) library is used to interact with these nodes safely.

## Database

### Schema

See [content/docs/database-schema.md](content/docs/database-schema.md) for detailed schema, column definitions, and relationships.

### Migrations

The service uses `node-pg-migrate` for database migrations. These migrations are located in `src/migrations/`. The service automatically runs the migrations when starting up.

#### Create a new migration

Migrations are created by running the create command:

```bash
yarn migrate create name-of-the-migration
```

This creates a migration file in the `src/migrations/` directory. This file MUST contain the migration setup and rollback procedures.

#### Manually applying migrations

If required, these migrations can be run manually.

To run them manually:

```bash
yarn migrate up
```

To roll them back manually:

```bash
yarn migrate down
```

## Getting Started

### Prerequisites

Before running the services, ensure you have the following installed:

- **Node.js**: Version 24.x or higher (LTS recommended)
- **Yarn**: Version 1.22.x or higher
- **Docker**: For running the PostgreSQL database

### Installation

1. Clone the repository:

```bash
git clone https://github.com/decentraland/catalyst.git
cd catalyst
```

2. Install dependencies:

```bash
yarn install
```

3. Build the project:

```bash
yarn build
```

### Configuration

Each service uses environment variables for configuration. See [docs/CONFIGURATION.md](docs/CONFIGURATION.md) for Content Server configuration options, and refer to each service's README for additional details.

### Running the Services

This is a monorepo containing two services:

- **Content Server** (`content/`): Entity storage and synchronization
- **Lambdas Server** (`lambdas/`): Utility APIs for clients

#### Running Content Server

```bash
yarn start:content
```

This starts the PostgreSQL database and the Content Server. For detailed configuration options, see [content/README.md](content/README.md).

#### Running Lambdas Server

The Lambdas Server requires a running Content Server:

```bash
yarn start:lambdas
```

For detailed configuration options, see [lambdas/README.md](lambdas/README.md).

## Testing

### Running Tests

Run all tests across both services:

```bash
yarn build
yarn test
```

Run tests for a specific service:

```bash
yarn test:content    # Content Server tests
yarn test:lambdas    # Lambdas Server tests
```

### Service-Specific Testing

Each service has additional testing options:

- **Content Server**: Supports separate unit and integration tests, CI mode, and debugging. See [content/README.md](content/README.md#run-unit-tests) for details.

- **Lambdas Server**: See [lambdas/README.md](lambdas/README.md#run-tests) for details.
